# Update README

## Purpose

Keep `README.md` current whenever the project gains a new feature, route, or dependency.

## Instructions

1. Review the code changes to identify:
   - What changed in the project.
   - How users can access or use the new feature or route.
   - Any new dependency that was added.
2. Update `README.md` with a concise description of what changed.
3. Explain how to use the new feature, including routes, commands, forms, or configuration when relevant.
4. Add any new dependencies to the README's dependency information.
5. Keep the README organized with these sections:
   - Project Overview
   - Features
   - Setup
   - Dependencies
6. Follow the existing README format if one exists. If the README uses different headings, preserve its style while adding or updating the required information in the closest matching sections.
7. Keep README entries accurate and avoid documenting behavior that is not implemented in the project.
